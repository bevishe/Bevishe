#反射
![](img/figure1.png)  
### 1.什么是反射？  
在运行状态中，对任何一个类，都能够获取这个类所有的属性和方法，对于任何一个对象，都能够调用它的任意一个方法和属性（包括私有的方法和属性）
，这种动态获取的信息以及动态调用对象的方法的功能就称为java语言的反射机制。通过反射，该类对我们来说是完全透明的，想要获取任何东西都可以。  


想要使用反射机制就必须要先获取该类的字节码文件对象（.class），通过字节码文件对象就能够获取该类中的所有信息（
方法，属性，类名，父类名，实现的所有接口等），每一个类对应着一个字节码文件也就对应着一个Class类型的对象，也就是
字节码文件对象。  

### 2.获取字节码文件对象的三种方法 
* 1.通过Class类中的静态方法forName  
通过Class类中的静态方法forName,直接获取到一个类字节码文件对象，此时该类还是源文件阶段，并没有变为字节码文件。
```java
Class class1 = Class.forName("全限定类名");
```  
* 2.当类被加载成class文件时，此时的Person类变成了.class，在获取该字节码对象，也就是获取自己，此类处于字节码阶段。   
```java
Class class2 = Person.class;
```  
* 3.当类已经创建出来实例，可以通过对象.getClass方法来获取字节码文件对象。
```java
Class class3 = p.getClass();
```  

### 3.通过反射可以获取到哪些信息？ 
* 1.通过字节码文件对象可以创建实例对象 
 ```java
Class class1 = Class.forName("com.cqupt.reflaction.Person");

        try {
            Person person1 = (Person) class1.newInstance();
            person1.setName("class1");
            person1.setAge(12);
            person1.printInfo();
        } catch (InstantiationException e) {
            e.printStackTrace();
        } catch (IllegalAccessException e) {
            e.printStackTrace();
        }

```

* 2.通过字节码文件对象来获取构造器
```java
        // 2.通过反射来获取构造器
        Class class2 = Class.forName("com.cqupt.reflaction.Person");
        // 获取的有参构造方法
        Constructor constructorArgs = class2.getConstructor(String.class,int.class);
        Person personArgs = (Person) constructorArgs.newInstance("classArgs",12);
        personArgs.printInfo();
        // 获取无参数的构造方法
        Constructor constructorNoArgs = class2.getConstructor();
        Person personNoArgs = (Person) constructorNoArgs.newInstance();
        personNoArgs.printInfo();
```

* 3.通过字节码文件对象来获取成员变量来使用 Field对象   
```java
        Class class3 = Class.forName("com.cqupt.reflaction.Person");
        Person personField = (Person) class3.newInstance();
        Field fieldName = class3.getDeclaredField("name");
        fieldName.setAccessible(true);
        fieldName.set(personField,"通过获取field来赋值的name");
        Field fieldAge = class3.getDeclaredField("age");
        fieldAge.setAccessible(true);
        fieldAge.setInt(personField,12);
        personField.printInfo();
```

* 4.通过字节码文件对象来获取成员方法
```java
        System.out.println("******************************************");
        Class class4 = Class.forName("com.cqupt.reflaction.Person");
        // 实例化
        Person personMethod = (Person) class4.newInstance();
        Method methodPrint = class4.getMethod("print");
        // 公共的方法
        methodPrint.invoke(personMethod);
        // 私有的方法
        Method methodPrintPri = class4.getDeclaredMethod("printPri", String.class);
        methodPrintPri.setAccessible(true);
        methodPrintPri.invoke(personMethod,"私有方法 ");
```

* 5.通过字节码文件对象获取类中所有的接口 
````java
Class[] getInterface(); 返回接口的字节码文件对象的数组
````

* 6.通过字节码文件对象来获取指定资源的输入流

InputStrem getResourceAsStream(String name);
返回的是一个InputStream对象,如果找不到带有该名称的资源返回null.

* 7.通过反射实现动态代理
动态代理：一种设计模式 


---
## 代理分为两种：核心思路是一个接口有两个子类，一个子类完成核心的业务操作，另外一个子类完成与核心业务有关的辅助性操作。

代理中涉及的角色：  
* 1.抽象角色： 为真实对象和代理对象提供一个公关的接口，一般是抽象类或者是接口  
* 2.代理角色： 代理角色内部含有对真实角色的引用，从而可以真正的操作真实的对象，同时代理角色提供了与真实角色相同的接口以便在任何时候都能代替真实对象。同时  
代理对象可以执行真实对象的操作时，附加其他的操作，相当于对真实对象的功能进行了扩展。   
* 3.真实角色： 最终引用的对象。   
---
* 1.静态代理：代理类在程序运行前就已经存在，这种代理称为静态代理。


* 2.动态代理 
实现动态代理必须要实现反射中的InvocationHandler接口，该接口中只实现了一个invoke（）方法，在代理实例上处理方法调用并返回结果。
